# PickHue — Chrome Web Store submission guide

Use this checklist when uploading to the [Developer Dashboard](https://chrome.google.com/webstore/devconsole).

## Before you upload

```bash
npm install
npm run package          # builds dist/ and creates pickhue-0.1.0.zip
npm run screenshots      # generates store/screenshots/*.png (optional)
```

Manual test on a fresh unpacked `dist/` load — see README checklist.

## Required URLs

| Field | Value |
|---|---|
| **Homepage** | https://pickhue.site |
| **Privacy policy** | https://pickhue.site/privacy |
| **Support** (optional) | https://github.com/alex-wilczewski/pickhue/issues |

> Deploy `store/privacy.html` to `https://pickhue.site/privacy` before submitting.
> Google requires a privacy policy URL even when you collect no data.

## Store listing copy

**Name:** PickHue

**Summary** (max 132 characters):

> Pick any color from any webpage with a precise magnifier. Copy HEX, RGB, or HSL. Recent colors, themes. Private — nothing leaves your device.

**Description:**

> PickHue is an elegant color picker for Chrome and Brave.
>
> Click the toolbar icon to open a clean panel right on the page. Hit **Select Color** to activate a circular magnifier that follows your cursor — click any pixel to copy it instantly.
>
> **Features**
> - Pixel-precise eyedropper with magnifier loupe
> - Copy as HEX, RGB, or HSL
> - Recent colors with one-click re-copy
> - System, light, and dark themes
> - Fully local — no accounts, no servers, no tracking
>
> **Privacy**
> PickHue does not collect or transmit personal data. Settings and recent colors are stored only in your browser.
>
> **Note**
> Color picking is not available on browser-internal pages (e.g. chrome://) or the Chrome Web Store, per browser security policy.

**Category:** Productivity

**Language:** English

**Pricing:** Free

## Privacy practices (dashboard form)

| Question | Answer |
|---|---|
| Single purpose | Color picking from web pages |
| Collects user data? | **No** (no transmission to developer servers) |
| Uses chrome.storage? | **Yes** — theme, format, recent colors (local/sync only) |
| Host permission justification | Required to inject the picker UI and sample pixel colors from the visible page when the user activates the eyedropper. PickHue only runs when the user clicks the toolbar icon. |

## Permission justifications (if prompted)

- **`activeTab`** — Access the current tab when the user clicks the extension icon.
- **`scripting`** — Inject the panel and eyedropper into the active page.
- **`storage`** — Save theme, color format, and recent colors locally.
- **`host_permissions: <all_urls>`** — Allow the eyedropper to run on pages the user chooses to pick colors from.

## Publisher name

The listing shows your **developer account name** by default. You can align branding with **PickHue** via:

- Store listing name: **PickHue**
- Homepage: **https://pickhue.site**

Using “PickHue” as the visible product name is standard; the legal publisher is still your Google developer account.

## Submit

1. Dashboard → **New item** → upload `pickhue-0.1.0.zip`
2. Fill listing, privacy URL, screenshots
3. Complete privacy practices questionnaire
4. **Submit for review** (typically 1–3 business days; broad host permissions may take longer)

## After approval

- Add the Chrome Web Store install link to https://pickhue.site
- Check off “Publish to Chrome Web Store” in README roadmap
